# HU MQ 操作手册

## 1. 登录

打开 HU MQ 监控后台，使用管理员账号登录。

## 2. 仪表盘

仪表盘展示全局概览：
- 集群数量、Topic 数量、消息总量、告警数量
- TPS 实时吞吐量曲线
- 消费延迟 (Lag) 柱状图

## 3. 集群管理

### 注册集群
1. 点击「注册集群」
2. 输入集群名称（如「生产集群」）
3. 输入 Kafka 连接地址（如 `kafka:9092`）
4. 确认后系统自动检测连接状态

### 查看集群详情
点击集群的「详情」查看：Broker 列表、Topic 数、分区数

## 4. Topic 管理

### 创建 Topic
1. 选择目标集群
2. 点击「创建 Topic」
3. 填写 Topic 名称、分区数、副本数
4. 确认创建

### 查看 Topic 详情
点击 Topic 的「详情」查看：分区列表、Leader/Replica/ISR 状态

### 删除 Topic
**注意：** 删除操作不可逆，请谨慎操作。

## 5. 消费组监控

选择集群后查看：
- 消费组列表及其订阅的 Topic
- 成员数量、状态 (Stable/Empty/Dead)
- 消费积压量 (Lag)，红色表示积压严重

## 6. 消息追踪

1. 选择集群
2. 输入 Topic、分区、起始 Offset
3. 点击「查询消息」查看消息内容

## 7. 告警中心

### 创建告警规则
1. 选择集群
2. 点击「创建规则」
3. 选择监控指标 (Lag/TPS/磁盘/Broker)
4. 设置条件（大于/等于）和阈值
5. 确认创建

### 查看告警事件
在「告警事件」标签页查看历史告警记录。

## 8. 权限管理

### 用户管理
管理员可以添加/删除用户，分配 admin 或 user 角色。

### ACL 规则
管理员可以为用户设置 Topic/消费组级别的读写权限。

## 9. 运维工具

### 分区重分配
选择集群和 Topic，执行分区重分配以均衡 Broker 负载。

## 10. 使用指引

内置使用指引帮助新人快速上手。
