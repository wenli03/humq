# HU MQ 最佳实践

## Topic 设计

### 命名规范

建议格式：`{业务线}.{子系统}.{事件类型}`

示例：
- `order.payment.success`
- `user.account.created`
- `inventory.stock.updated`

### 分区规划

| 预估 TPS | 建议分区数 |
|----------|-----------|
| < 1,000 | 1 |
| 1,000 - 10,000 | 3 |
| 10,000 - 100,000 | 10-20 |
| > 100,000 | 30+ |

- 分区数应为 Broker 数的整数倍
- 预留 20% 余量应对流量增长

### 保留策略

- 业务日志：7 天
- 事件溯源：30 天
- 实时通知：1 天

## 消费者最佳实践

- 一个消费组处理一类业务
- 消费者数量 <= 分区数
- 定期检查消费 Lag，保持在合理范围
- 实现消费幂等性，防止重复消费

## 告警阈值建议

| 指标 | 警告 | 严重 |
|------|------|------|
| 消费 Lag | > 1,000 | > 10,000 |
| Broker 磁盘 | > 70% | > 85% |
| TPS 下降 | 下降 30% | 下降 50% |
| Broker 离线 | 立即 | 立即 |

## 安全建议

- 生产环境修改默认 JWT 密钥
- 定期更换管理员密码
- 使用 ACL 限制 Topic 访问
- 启用 HTTPS
- 定期备份 PostgreSQL 数据
